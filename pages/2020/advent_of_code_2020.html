Advent of code 2020

What is it

Each year since 2015, Advent of Code is a series of 25 puzzles in December.

The goal of a puzzle is to compute the correct output by following the instruction and using a given text input. There are no restrictions on the language or techniques you can use.

The event can be seen as competetive programming: the puzzles are released at the same time worldwide and a leaderboard system tracks the fastest answers. The first to get the correct answer receives 100 points, the second gets 99 points and the 100th gets 1 point (only the first 100 to correctly answer the problem get points.).

Previous years

I participated in 2017, 2018 and 2019 using Python. It was a great experience and I got to learn a lot about algorithms and Python. In 2018 and 2019, I even got in the global leaderboard (look me up, I'm user #230381).

Python is an excellent language for solving these puzzles quickly because it is very fast to write Python scripts and you typically do not need very high execution speed in AoC.

Advent of Code is a great tool to learn more about languages and algorithms because it provides exercises which are definitely necessary to get better (like everything in life, you can't get better at it without actual practice). Additionally, the Christmas theme and seeing everyone doing these problems are good sources of motivation.

Using Zig to solve 2020

This year, I didn't want to wake up at 6 AM every day in December to solve Advent of Code. Using Python would not have been interesting because I already know the language well and I wasn't trying to get in the leaderboard.

I decided to use Zig to learn the language. Zig is supposed to be "like C but better while staying simple".

The language is still immature (latest version is 0.7.1) but it already works quite well. There is one documentation page but it only covers the basis. To understand how to use the stdlib, the best method is to look directly at its source code. Each file has some unit tests that also serve as examples.

After using Zig to solve these simple problems, I find that I like the language a lot. I hope its developement will continue and that it will get more traction in the future.
